Server-Side Web Application Development: Online Cookbook

Case Study
Languages: PHP, MySQL, HTML5, CSS3, Git

Overview

As the solo designer and developer, I created an end-to-end server-side web application that allows users to browse, search, and filter a library of recipes. This project was part of my course, IDM232, which focused on equipping students with the skills to develop secure, dynamic, and data-driven web applications. From handling user input to database management and building RESTful APIs, I applied foundational web development principles and best practices throughout the process.

This course allowed me to expand my skills beyond front-end design and focus on developing secure, scalable server-side applications. The primary challenge was effectively integrating user input handling and database interactions into a cohesive and fully functional application.

Process and Insight

Technology Stack

I used PHP for server-side scripting, MySQL for database management, and CSS for a responsive and accessible user experience.


Design and Architecture:

I began by designing a simple, user-friendly interface for the recipe library. The application allowed users to view recipes, search through them by ingredients or categories, and filter results based on various criteria. On the back end, I implemented structured code with proper error handling, modularity, and security practices.


Research

I researched best practices for building secure, scalable web applications. I explored how to effectively connect front-end and back-end technologies, ensuring smooth data flow and responsiveness. I also reviewed existing online recipe platforms to understand user expectations and features to include in my project.

Challenge and Solution

Challenge

I encountered issues with database connectivity and learned how to handle errors effectively. Ensuring the front-end and back-end worked seamlessly together also required careful planning.

Solution

I used a combination of research, online resources, and trial and error to overcome challenges. By incorporating security best practices and using prepared statements, I protected the database from SQL injection attacks. Error handling was implemented to ensure the application remained stable and user-friendly.

The Results

The final product was a fully functional online cookbook that allowed users to interact with a large recipe database. I was able to implement all the required features: handling user input securely, connecting to a MySQL database, managing authentication, and building a RESTful API. The web application was stable, secure, and offered a smooth user experience.
